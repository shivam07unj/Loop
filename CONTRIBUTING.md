## Branches

| Branch | Purpose |
|---|---|
| `main` | Stable, reviewed code only |

## Workflow

1. Branch off `dev` â€” `git checkout -b feature/your-feature dev`
2. Make your changes with clear, descriptive commits
3. Open a Pull Request into `dev` with a description of what changed
4. At least one team member must review before merging

## Commit Convention

```
feat: add risk detection ML model
fix: resolve Slack webhook timeout
docs: update README with setup steps
refactor: clean up alert engine logic
```
